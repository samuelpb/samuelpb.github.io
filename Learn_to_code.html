Should you learn to code?
Yes. If you are asking yourself this question, then I completely recommend taking the time to learn at least a little bit. There are plenty of resources to help you do so. Even if it not something you do forever, a little background on how coding works will never hurt.

Should you become a programmer?
This is a completely different question. Coding requires a different way of thinking, a logical one, that revolves around some abstractness. Being able to understand and keep track of tasks in your head with visualizations can be vital. So can understanding math and logic, depending on the type of programmer you would want to be.

Types of Progamming
Personally, I consider there to be three types of programming: Design, Algorithmic, and Systems. Design is less math , but instead logic and creativity around designing a web page or front screen of an application. There are many tools to help do this, so the amount of deep coding logic and thinking is minimal, and for that reason this is the easiest path into programming. Second is algorithmic. This requires some math and logic background, and can range from statistics and soritng algorithms to machine learning and artificial intelligence. This is what I consider the most interesting, but by far the most difficult programming. Lastly, systems program revolves around design and support of applications, systems, and structures for a company, which is extrememly common (and mostly what I have dealt with in both my jobs so far). This requires a fair bit of logic and math, depending on the system, and also requires a fair bit of systems understanding, especially around object-oriented proramming.

So what language should you learn?
Well, it depends on which type of programming. For front end design, javascript is probably the best. I personally am not the person to turn to for front-end, as I have only a little experience with it. For algorithmic, python is very popular with its packages around machine learning, and java is never a bad choice either. And for systems, I highly recommend java. Learning the coding basics, along with packages and object-oriented programming is extrememly useful. In fact, if you are not sure which type of programming you like, still start with Java. Learning the way of thinking behind programming is the most important.

Where do you start?
Anywhere. I know Java the best, I understand python very well especially for beginner programmers, and I am willing to work with either if you would like some tutoring. But, there are so many websites and courses online to start learning. Udemy, Codeacademy, etc. For anything front end, a site like that would help better than I would. But overall, it never hurts to get started, and you can find which style works best for you.
